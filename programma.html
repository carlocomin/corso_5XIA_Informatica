Programma del corso (ITST – 5ª)

Panoramica

  • Obiettivo generale: passare dalla modellazione dei dati alla loro gestione in una piccola applicazione web con database relazionale.
  • Metodo: rigore formale, esempi guidati, micro-verifiche e richiami operativi nei punti chiave.
  • Progressione: modello relazionale, progettazione concettuale, PHP lato server, MySQL con prepared statements, normalizzazione.
  • Output atteso: capacità di modellare, tradurre, interrogare e usare un database in una web app didattica semplice ma corretta.

Come leggere il materiale

  • Nel modello: definizioni teoriche e vincoli del formalismo.
  • Nella traduzione: passaggio da schema concettuale a schema relazionale.
  • Nel DBMS reale: SQL, PDO, MySQL e scelte implementative essenziali.
  • Box verdi: esempi corretti o esiti validi.
  • Box arancioni: avvertenze, errori tipici, controesempi.
  • Box blu: note operative, suggerimenti, definizioni rapide.
  • Pulsanti Mostra/Nascondi per approfondimenti o soluzioni a comparsa.
