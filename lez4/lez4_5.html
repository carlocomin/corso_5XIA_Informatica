Copertura minimale: regole 1–2–3 e procedura

Le tre trasformazioni standard (tutte “sound”)

  • Regola 1: scomposizione del conseguente (RHS singolo).
  • Regola 2: eliminazione di attributi superflui a sinistra.
  • Regola 3: eliminazione di dipendenze ridondanti.

“Sound” = non cambiano il contenuto informativo (non cambiano la chiusura).

Intuizione: stai “snellendo” la descrizione dei vincoli, non i vincoli stessi.

Regola 1 — scomposizione del conseguente

Da:

X → (A B C)

a:

X → A
X → B
X → C

Regola 2 — attributi superflui a sinistra

In una DF X → A (RHS singolo), un attributo b ∈ X è superfluo se:

A ∈ (X − {b})⁺

Quindi il test si fa calcolando una chiusura.

Intuizione: se riesci comunque a ottenere A senza usare b, allora b non “aggiunge potere” e può essere tolto.

Regola 3 — dipendenze ridondanti

Una DF X → A è ridondante se, tolta dall’insieme:

A ∈ X⁺ calcolata rispetto a F' = F \\ {X → A}

Intuizione: se le altre regole permettono già di arrivare a A partendo da X, allora quella DF era solo una ripetizione.

Procedura standard (riassunto)

  1. RHS singolo (Regola 1)
  2. Elimina superflui a sinistra (Regola 2)
  3. Elimina DF ridondanti (Regola 3)

Ripeti finché non ci sono ulteriori semplificazioni.

Esempio guidato completo

Partiamo da:

F = { A → BC,  B → C,  A → B }
1) Regola 1
F1 = { A → B,  A → C,  B → C }
2) Regola 2

Qui la parte sinistra ha un solo attributo nelle DF, quindi non c’è nulla da eliminare.

3) Regola 3

Controllo se A → C è ridondante:

  • Rimuovo A → C: resta {A → B, B → C}.
  • Calcolo A⁺: con A→B ottengo B, poi con B→C ottengo C.

Quindi A → C è implicata dalle altre e si elimina.

Fm = { A → B, B → C }

Micro-esercizio

Dato F = { AB → C, A → C }, l’attributo B è superfluo nella DF AB → C?

Test: calcolo A⁺. Poiché A → C è in F, allora C ∈ A⁺.

Quindi sì: B è superfluo e AB → C si riduce a A → C.
