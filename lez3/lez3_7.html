Funzioni di utilità sugli array

Queste funzioni sono prima di tutto strumenti generali di PHP. In questo modulo le useremo sia negli esercizi introduttivi sia, piu avanti, su input gia validati e risultati di query.

array_sum(array $valori): somma di tutti gli elementi

Utile quando devi sommare valori numerici gia raccolti in un array.

<?php
$prezzi = [9.99, 14.50, 3];
$totale = array_sum($prezzi);

echo "Totale carrello: €" . number_format($totale, 2, ',', '.');
?>

Negli esercizi puo servire per somme e medie; con una query potresti usarla dopo aver estratto una colonna numerica.

array_map(callable $f, array $a): applica una funzione a tutti gli elementi

Utile per trasformazioni senza scrivere ogni volta un ciclo esplicito.

<?php
$voti = [18, 23, 30];
$stati = array_map(function($voto) {
  return $voto >= 18 ? "Promosso" : "Insufficiente";
}, $voti);

print_r($stati);
?>

La stessa idea puo servire per formattare nomi o etichette letti da MySQL prima di stamparli in HTML.

array_filter(array $a, ?callable $f = null): filtra gli elementi

Serve a tenere solo gli elementi che rispettano una condizione.

<?php
$numeri = [10, 5, 0, -2, 7];
$positivi = array_filter($numeri, function($n) {
  return $n > 0;
});

print_r($positivi);
?>

Negli esempi base filtra numeri o stringhe; dopo una query puo servire per lavorare solo su una parte del risultato.

in_array(mixed $needle, array $haystack): cerca un valore

Con il terzo parametro true esegue il confronto stretto.

<?php
$classiConsentite = ["5BIA", "5AIA", "5CINF"];
if (in_array("5BIA", $classiConsentite, true)) {
  echo "Classe valida";
}
?>

Passare true evita confronti ambigui quando i dati arrivano da form o query.

array_merge(array ...$arrays): unisce più array

Utile per unire dati base e dati aggiuntivi in un solo array.

<?php
$datiBase = ["nome" => "Ada"];
$datiExtra = ["classe" => "5BIA", "email" => "ada@example.org"];
$studente = array_merge($datiBase, $datiExtra);

print_r($studente);
?>

Con chiavi stringa, l'ultimo valore vince se una chiave e presente in entrambi gli array.

array_key_exists(string|int $chiave, array $array): verifica la presenza di una chiave

Restituisce true anche se il valore associato e null.

<?php
$utente = [
  "nome" => "Luca",
  "ruolo" => null,
];

var_dump(array_key_exists("ruolo", $utente)); // true
var_dump(isset($utente["ruolo"]));             // false
?>

E utile quando un campo puo essere presente ma vuoto, per esempio dopo una validazione.

Combina le funzioni per pipeline compatte

Le funzioni sugli array diventano potenti quando le combini in sequenza.

<?php
$righe = [
  ['nome' => 'ada', 'classe' => '5BIA'],
  ['nome' => 'luca', 'classe' => '4BIA']
];

$soloQuinte = array_filter($righe, function($riga) {
  return $riga['classe'] === '5BIA';
});
$nomi = array_map(function($riga) {
  return strtoupper($riga['nome']);
}, $soloQuinte);

print_r($nomi);
?>

Questa logica e perfetta quando vuoi pulire o trasformare dati prima di stamparli nella view.

Da ricordare

  • Prima capisci la funzione con un array semplice.
  • Poi riusi la stessa idea su array piu complessi, compresi quelli letti dal database.
  • Se il codice diventa meno leggibile di un foreach, un ciclo esplicito puo restare la scelta migliore.
