Tipi di dato fondamentali e array

Obiettivi

  • Ripassare i datatype fondamentali piu usati in PHP.
  • Capire gli array come struttura chiave del linguaggio.
  • Riconoscere quando usare array indicizzati e quando usare array associativi.
  • Prepararsi a leggere e stampare risultati provenienti da MySQL.

Scalari: stringhe, numeri, boolean, null

<?php
$utente = "Giulia";
$eta = 19;
$media = 7.5;
$iscritta = true;
$telefono = null;

var_dump($utente, $eta, $media, $iscritta, $telefono);
?>
  • Questi sono i tipi di base che userai piu spesso negli script introduttivi.
  • null rappresenta l'assenza di valore ed e molto comune nei dati opzionali.
  • var_dump() e utile per capire il tipo reale di un dato durante il debug.

Cast e lettura sicura dei dati

<?php
$quantita = (int) ($_POST['quantita'] ?? 0);
$prezzo = (float) ($_POST['prezzo'] ?? 0);
$attivo = (bool) ($_POST['attivo'] ?? false);
?>
  • Il cast esplicito rende piu chiaro come userai il dato.
  • Quando i valori arrivano da un form, prima chiarisci il tipo e poi li validi o li elabori.
  • Questo stesso approccio verra riusato quando passeremo valori a PDO.

Perche gli array sono centrali in questo modulo

In PHP moltissimi dati applicativi vengono gestiti come array:

  • campi ricevuti da form;
  • righe restituite da una query;
  • liste di risultati da mostrare in HTML.

Per questo il modulo dedica piu spazio agli array che ad altri aspetti del linguaggio.

Array indicizzati: liste ordinate

<?php
$classi = ["5BIA", "5AIA", "5CINF"];
$classi[] = "5DINF";

foreach ($classi as $classe) {
    echo $classe . "<br>";
}
?>
  • Usa un array indicizzato quando l'ordine conta.
  • Gli elementi vengono letti tramite indice: 0, 1, 2...
  • La sintassi [] aggiunge un elemento in coda.

Array associativi: una riga con campi nominati

<?php
$studente = [
  'id' => 1,
  'nome' => 'Giulia',
  'classe' => '5BIA'
];

echo $studente['nome'];
?>
  • Qui ogni dato ha un nome semantico: id, nome, classe.
  • Questo rende il codice piu leggibile rispetto a un array solo numerico.

Questo formato e molto vicino a una riga di tabella letta da MySQL con PDO::FETCH_ASSOC.

Lista di righe: l'array che ritorna una query

<?php
$stmt = $pdo->query("SELECT id, nome, classe FROM studenti");
$righe = $stmt->fetchAll(PDO::FETCH_ASSOC);

foreach ($righe as $riga) {
    echo $riga['nome'] . " - " . $riga['classe'] . "<br>";
}
?>

Qui hai un array di array associativi: e il formato tipico dei risultati SQL in questo modulo.

Array annidati e accesso ai dati

<?php
$studenti = [
  ['nome' => 'Ada', 'classe' => '5BIA'],
  ['nome' => 'Alan', 'classe' => '5BIA']
];

echo $studenti[0]['nome']; // Ada
?>
  • Indice esterno: scegli la riga.
  • Chiave interna: scegli il campo della riga.
  • Questa struttura compare spesso sia negli esercizi sia nelle pagine che leggono il database.

Buone pratiche utili per il database

  • Usa chiavi chiare negli array associativi: devono corrispondere ai nomi dei campi.
  • Controlla l'esistenza di una chiave con ?? o array_key_exists().
  • Quando stampi valori da un array in HTML, usa htmlspecialchars().
  • Quando ricevi dati da form, fai cast e validazione prima di costruire l'array da passare alla query.

Checklist finale

  • Datatype scalari = valori singoli come stringhe, numeri, boolean e null.
  • Array indicizzato = lista.
  • Array associativo = record con campi nominati.
  • Risultato di query = lista di record.
