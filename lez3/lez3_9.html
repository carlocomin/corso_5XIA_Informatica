Variabili d'ambiente e superglobali PHP

Le superglobali sono array speciali sempre disponibili in ogni punto dello script. Nel nostro modulo servono soprattutto per leggere input utente, validarlo e poi usarlo in query parametrizzate verso MySQL.

Mappa didattica della slide

  • Nucleo del modulo: $_GET, $_POST, $_SERVER.
  • Passaggio chiave: leggere l'input, validarlo, usarlo in una query parametrizzata.
  • Le altre superglobali vanno conosciute, ma qui restano sullo sfondo.

$_GET — Parametri in query string

Contiene i parametri passati via URL (es. pagina.php?nome=Anna&eta=17). Non modifica mai lo stato del server ed è usato per filtri, ricerche e navigazione. I valori arrivano come stringhe: valida e sanifica sempre.

<?php
// pagina.php?nome=Anna&eta=17
$nome = $_GET['nome'] ?? 'ospite';
$eta  = $_GET['eta'] ?? 'n/d';

echo "Ciao $nome, ho ricevuto età: $eta";
?>

Esempio diretto e volutamente semplice: serve a capire la meccanica di $_GET prima della validazione.

$_POST — Dati inviati da form

Raccoglie i campi di un modulo con method="POST". È il canale piu naturale quando l'utente invia dati da salvare o elaborare sul server.

<form action="salva-contatto.php" method="POST">
  <input name="email" type="email" required>
  <textarea name="messaggio" required></textarea>
  <button type="submit">Invia</button>
</form>

<?php // salva-contatto.php
$email = $_POST['email'] ?? null;
$msg   = $_POST['messaggio'] ?? null;

if (!$email || !$msg) {
  exit('Dati mancanti');
}

echo "Grazie, contatteremo $email. Messaggio: $msg";
?>

Anche qui partiamo dal caso base: prima capisci che cosa arriva in $_POST, poi aggiungi validazione e query parametrizzate.

$_SERVER — Contesto della richiesta

Contiene informazioni utili sulla richiesta in corso. Nel modulo ci interessa soprattutto capire se la pagina e stata aperta in GET o inviata in POST.

<?php
if (($_SERVER['REQUEST_METHOD'] ?? 'GET') === 'POST') {
  echo "Il form e stato inviato.";
} else {
  echo "Mostro il form.";
}
?>

Questa variabile ti aiuta a distinguere tra la pagina che mostra il form e la pagina che lo elabora.

Le superglobali centrali del modulo

  • $_GET: filtri e parametri di lettura.
  • $_POST: dati da form da validare prima del salvataggio.
  • $_SERVER: capire se la richiesta e GET o POST e ottenere contesto utile.

Nel percorso del modulo queste tre sono il ponte diretto tra pagina HTML, logica PHP e database.

Validazione essenziale con filter_input

Quando il meccanismo base e chiaro, il passo successivo e validare l'input. Per il nostro livello basta capire il principio: scegli la sorgente, leggi il campo, applichi un filtro.

  • FILTER_SANITIZE_SPECIAL_CHARS: prepara una stringa da mostrare in HTML.
  • FILTER_VALIDATE_INT e FILTER_VALIDATE_EMAIL: verificano se un valore ha il formato atteso.
  • Se il valore non e valido, non lo usi: mostri un messaggio o blocchi l'operazione.
<?php
$nome = filter_input(INPUT_GET, 'nome', FILTER_SANITIZE_SPECIAL_CHARS);
$eta  = filter_input(INPUT_GET, 'eta', FILTER_VALIDATE_INT, [
  'options' => ['min_range' => 1, 'max_range' => 120]
]);

$etaDescrizione = $eta === false ? 'età non valida' : ($eta === null ? 'nessuna età' : $eta . ' anni');
echo "Ciao $nome, ho ricevuto $etaDescrizione";
?>

filter_input_array() esiste ed e utile su piu campi insieme, ma per il modulo basta padroneggiare bene i casi semplici.

Altre superglobali da conoscere, senza approfondirle ora

  • $_FILES: descrive i file caricati con un form e richiede enctype="multipart/form-data".
  • $_COOKIE: legge preferenze salvate nel browser, ma non e adatto a dati sensibili.
  • $_SESSION: mantiene dati lato server tra una richiesta e l'altra dopo session_start().

Sono utili da conoscere, ma non sono il centro di questo modulo introduttivo.

Dal form al prepared statement

Questo e il collegamento che ci interessa di piu nel modulo:

<?php
$nome = filter_input(INPUT_POST, 'nome', FILTER_SANITIZE_SPECIAL_CHARS);
$classe = filter_input(INPUT_POST, 'classe', FILTER_SANITIZE_SPECIAL_CHARS);

if ($nome && $classe) {
  $stmt = $pdo->prepare(
    "INSERT INTO studenti (nome, classe)
     VALUES (:nome, :classe)"
  );

  $stmt->execute([
    'nome' => $nome,
    'classe' => $classe
  ]);
}
?>

La superglobale da sola non basta: il passaggio corretto e sempre leggere, validare, passare i valori alla query parametrizzata.
