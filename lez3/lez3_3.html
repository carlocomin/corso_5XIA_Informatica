Setup ambiente: PHP + MySQL

Obiettivo della sessione

  • Avere un ambiente locale con PHP e MySQL funzionanti.
  • Preparare una connessione PDO riutilizzabile.
  • Arrivare non solo a "Connessione OK", ma al primo contatto con una query parametrizzata.

Cosa serve davvero

  • Server web locale o bundle equivalente (XAMPP, WAMP, MAMP, LAMP).
  • PHP con estensione pdo_mysql attiva.
  • Un'istanza MySQL o MariaDB raggiungibile in locale.

Per il modulo conta il risultato finale: browser raggiungibile, DB pronto, PDO disponibile.

Configurazione minima

  • Avvia il server web e MySQL.
  • Individua la cartella da cui il browser legge i file PHP.
  • Verifica che http://localhost risponda.
  • Controlla da terminale la versione PHP con php -v.

Se PHP non vede l'estensione pdo_mysql, le slide successive sul database non potranno funzionare.

Prepara il database e la tabella guida del modulo

CREATE DATABASE corso_php CHARACTER SET utf8mb4;
CREATE USER 'studente'@'localhost' IDENTIFIED BY 'password_sicura';
GRANT ALL PRIVILEGES ON corso_php.* TO 'studente'@'localhost';

USE corso_php;

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  classe VARCHAR(20) NOT NULL
);

Nel modulo ricorreremo spesso alla tabella studenti: averla pronta rende piu chiari esempi, query e prepared statements. Qui compare anche l'utente studente, cosi la parte SQL e l'esempio PDO restano coerenti tra loro.

Prima connessione PDO

<?php
$dsn = 'mysql:host=localhost;dbname=corso_php;charset=utf8mb4';
$user = 'studente';
$pass = 'password_sicura';

try {
    $pdo = new PDO($dsn, $user, $pass, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]);
    echo "Connessione OK";
} catch (PDOException $e) {
    echo "Errore: " . htmlspecialchars($e->getMessage());
}

Sostituisci $user e $pass con le credenziali reali del tuo ambiente locale.

Questa connessione e la base di tutto il modulo: la riuseremo per leggere e inserire dati.

Perche usare PDO in questo corso

  • Un'unica interfaccia chiara per lavorare con il database.
  • Supporto diretto ai prepared statements.
  • Codice piu leggibile e riutilizzabile rispetto a query costruite a mano.

Primo esempio di prepared statement

<?php
$stmt = $pdo->prepare(
    "SELECT id, nome, classe
     FROM studenti
     WHERE classe = :classe"
);

$stmt->execute([
    'classe' => '5BIA'
]);

$righe = $stmt->fetchAll();
?>

Il punto chiave non e memorizzare i metodi, ma capire la sequenza: prepare, poi execute, poi lettura del risultato.

Checklist finale

  • PHP e MySQL attivi.
  • Database corso_php creato.
  • Tabella studenti pronta.
  • Connessione PDO verificata.
  • Idea chiara di cosa sia un prepared statement.
