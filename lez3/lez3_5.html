Sintassi base e tipi

Obiettivi

  • Prendere confidenza con variabili, commenti e regole di base della sintassi PHP.
  • Conoscere i tipi primitivi principali (stringhe, numeri, boolean, null) e come li usi nel codice e nelle pagine HTML.
  • Evitare errori comuni: concatenazione, comparazioni deboli vs strette, interpolazione e gestione degli input mancanti.

Questa base è indispensabile prima di affrontare controlli di flusso e strutture dati: solo con tipi e sintassi chiari puoi leggere e manutenere il codice in modo affidabile.

Sintassi minima: variabili e commenti

<?php
$nome = "Giulia";       // variabile: il simbolo $ è obbligatorio
$eta  = 18;              // numeri senza virgolette

// commento su una linea
/* commento su più linee */

// case-sensitive su variabili: $Nome è diverso da $nome
?>
  • Le istruzioni terminano con ;.
  • Usa nomi chiari (snake_case o camelCase) e indenta il codice come negli esempi: il file sarà leggibile anche a distanza di tempo o da altri compagni.
  • Ricorda che PHP è case-sensitive sulle variabili ma non sui comandi echo/Echo; mantieni comunque uno stile coerente tutto minuscolo.

Stringhe: singoli vs doppi apici

<?php
$nome = "Giulia";

$double = "Ciao $nome";           // interpolazione variabili
$single = 'Ciao $nome';            // stringa letterale
$concat = "Ciao " . $nome . "!"; // concatenazione con .

echo $double;  // Ciao Giulia
?>
  • I doppi apici espandono variabili e sequenze speciali (\n per andare a capo); i singoli mantengono tutto letterale, utile per stringhe statiche o query SQL.
  • . è l'operatore di concatenazione (non +) e può concatenare anche numeri, che verranno convertiti in stringa.
  • Per sicurezza lato HTML, combina con htmlspecialchars() prima di mostrare input utente così da evitare injection o rotture del layout.

Numeri e booleani

<?php
$intero = 42;         // int
$decimale = 3.14;     // float
$vero = true;         // boolean
$falso = false;

// attenzione alle comparazioni deboli vs strette
var_dump(0 == false);   // true (confronto debole)
var_dump(0 === false);  // false (tipi diversi, confronto stretto)
?>
  • Preferisci gli operatori stretti === e !== quando confronti valori user-facing, perché controllano sia il valore sia il tipo.
  • var_dump() mostra tipo e valore: utile per debug rapido, ma rimuovilo dal codice consegnato.
  • Gli interi e i float vengono convertiti automaticamente quando li sommi o dividi, ma attenzione ai decimali (es. 0.1 + 0.2) per gli arrotondamenti binari.

Null, coalescenza e default

<?php
$telefono = null;
$telefono = $_POST['telefono'] ?? null; // prende POST se esiste, altrimenti null

// valori di default con l'operatore ??
$ruolo = $_POST['ruolo'] ?? 'studente';
?>
  • null rappresenta l'assenza di valore: differenzia tra "campo non inviato" e stringa vuota.
  • L'operatore di coalescenza ?? evita notice su indici non definiti e imposta default leggibili senza dover annidare isset().
  • Quando ricevi dati da form, usa ?? per prevenire warning e mostrare messaggi chiari all'utente.

Cast e conversioni rapide

<?php
$quantita = "5";         // stringa
$quantitaInt = (int) $quantita; // 5

$flag = "1";
if ((bool) $flag) {
  echo "vero";
}
?>
  • Casting esplicito rende chiaro come vuoi usare il dato (utile prima di salvare su DB) e riduce conversioni implicite imprevedibili.
  • Combina cast + validazione server-side per evitare input non coerenti: prima validi, poi casti, poi salvi.

In sintesi

  • Usa doppi apici per interpolare, singoli quando vuoi testo letterale.
  • Preferisci ===/!== per confronti sicuri tra tipi diversi.
  • Gestisci valori opzionali con ?? e assegna default espliciti per evitare notice e comportamenti inattesi.
  • Casta e valida i dati in ingresso prima di usarli: l'applicazione sarà più sicura e prevedibile.
