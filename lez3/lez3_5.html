Sintassi base e tipi

Obiettivi

  • Prendere confidenza con variabili, commenti e regole di base della sintassi PHP.
  • Conoscere i tipi primitivi principali (stringhe, numeri, boolean, null) e come li usi.
  • Evitare errori comuni: concatenazione, comparazioni deboli vs strette, interpolazione.

Questa base è indispensabile prima di affrontare controlli di flusso e strutture dati.

Sintassi minima: variabili e commenti

<?php
$nome = "Giulia";       // variabile: il simbolo $ è obbligatorio
$eta  = 18;              // numeri senza virgolette

// commento su una linea
/* commento su più linee */

// case-sensitive su variabili: $Nome è diverso da $nome
?>
  • Le istruzioni terminano con ;.
  • Usa nomi chiari (snake_case o camelCase) e indenta il codice come negli esempi.

Stringhe: singoli vs doppi apici

<?php
$nome = "Giulia";

$double = "Ciao $nome";           // interpolazione variabili
$single = 'Ciao $nome';            // stringa letterale
$concat = "Ciao " . $nome . "!"; // concatenazione con .

echo $double;  // Ciao Giulia
?>
  • I doppi apici espandono variabili; i singoli mantengono il testo letterale.
  • . è l'operatore di concatenazione (non +).
  • Per sicurezza lato HTML, combina con htmlspecialchars() prima di mostrare input utente.

Numeri e booleani

<?php
$intero = 42;         // int
$decimale = 3.14;     // float
$vero = true;         // boolean
$falso = false;

// attenzione alle comparazioni deboli vs strette
var_dump(0 == false);   // true (confronto debole)
var_dump(0 === false);  // false (tipi diversi, confronto stretto)
?>
  • Preferisci gli operatori stretti === e !== quando confronti valori user-facing.
  • var_dump() mostra tipo e valore: utile per debug.

Null, coalescenza e default

<?php
$telefono = null;
$telefono = $_POST['telefono'] ?? null; // prende POST se esiste, altrimenti null

// valori di default con l'operatore ??
$ruolo = $_POST['ruolo'] ?? 'studente';
?>
  • null rappresenta l'assenza di valore.
  • L'operatore di coalescenza ?? evita notice su indici non definiti e imposta default leggibili.

Cast e conversioni rapide

<?php
$quantita = "5";         // stringa
$quantitaInt = (int) $quantita; // 5

$flag = "1";
if ((bool) $flag) {
  echo "vero";
}
?>
  • Casting esplicito rende chiaro come vuoi usare il dato (utile prima di salvare su DB).
  • Combina cast + validazione server-side per evitare input non coerenti.

In sintesi

  • Usa doppi apici per interpolare, singoli quando vuoi testo letterale.
  • Preferisci ===/!== per confronti sicuri tra tipi diversi.
  • Gestisci valori opzionali con ?? e assegna default espliciti.
