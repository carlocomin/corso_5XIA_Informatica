Primo script PHP

Obiettivi veloci

  • Scrivere un file PHP mescolando logica e HTML in modo ordinato.
  • Stampare output dinamico lato server.
  • Preparare il terreno per mostrare in pagina anche dati presi da MySQL.

Struttura minima di un file PHP

<!DOCTYPE html>
<html lang="it">
<head>...</head>
<body>
  <h1>Ciao dal server!</h1>
  <p>Oggi e: <?php echo date('d/m/Y'); ?></p>
</body>
</html>

Il browser non riceve il codice PHP: riceve solo l'HTML che PHP ha prodotto.

Output dinamico e valori variabili

<?php
$nome = "Alice";
$classe = "5BIA";
?>

<h2>Benvenuta <?= htmlspecialchars($nome) ?></h2>
<p>Classe: <?= htmlspecialchars($classe) ?></p>

htmlspecialchars() va usato ogni volta che stampi dati che potrebbero arrivare da form, URL o database.

Dal PHP all'HTML

<?php
$studenti = ["Ada", "Alan", "Grace"];
?>

<ul>
  <?php foreach ($studenti as $studente): ?>
    <li><?= htmlspecialchars($studente) ?></li>
  <?php endforeach; ?>
</ul>

La logica prepara i dati; l'HTML descrive come visualizzarli.

Lo stesso schema con dati dal database

<?php
$stmt = $pdo->prepare(
    "SELECT nome, classe
     FROM studenti
     WHERE classe = :classe"
);
$stmt->execute(['classe' => '5BIA']);
$studenti = $stmt->fetchAll();
?>

<ul>
  <?php foreach ($studenti as $studente): ?>
    <li>
      <?= htmlspecialchars($studente['nome']) ?>
      -
      <?= htmlspecialchars($studente['classe']) ?>
    </li>
  <?php endforeach; ?>
</ul>

Idea chiave: cambia la sorgente dei dati, non il principio con cui li stampi in HTML.

Debug rapido

<?php
error_reporting(E_ALL);
ini_set('display_errors', 1);

var_dump($studenti);
// exit('stop');
?>
  • var_dump() aiuta a capire se hai una stringa, un array o il risultato di una query.
  • exit() ti ferma in un punto preciso quando vuoi controllare lo stato del programma.
  • Questi strumenti vanno bene in locale; non devono restare nel codice finale da consegnare.

Cosa fissare da questa slide

  • PHP produce HTML lato server.
  • Una lista HTML puo essere alimentata prima da un array, poi da una query MySQL.
  • Stampare dati in pagina e il punto di arrivo naturale del lavoro fatto con PDO e query parametrizzate.
