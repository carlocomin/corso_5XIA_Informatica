Ripasso e flusso richiesta/risposta

Domanda guida

Che cosa succede davvero quando un utente compila un form, preme invio e vede una pagina aggiornata con dati presi da MySQL?

Questa slide serve a fissare il percorso del dato prima di scrivere codice PHP e query parametrizzate.

I tre attori del flusso

  • Browser: mostra HTML/CSS, raccoglie input dal form e invia la richiesta.
  • PHP: riceve la richiesta, legge i dati, li valida, prepara la query e costruisce la risposta.
  • MySQL: conserva i dati e risponde alle query eseguite da PHP.

Il browser non esegue PHP: riceve solo l'HTML finale generato sul server.

Il percorso del dato

  1. L'utente apre una pagina PHP o invia un form.
  2. PHP legge i parametri con $_GET o $_POST.
  3. PHP valida i valori ricevuti.
  4. PHP usa PDO per parlare con MySQL.
  5. La query legge o scrive dati, meglio se con prepared statement.
  6. PHP stampa il risultato dentro HTML.

Schema mentale del modulo: form -> input PHP -> PDO -> MySQL -> HTML.

GET e POST: cosa cambia per noi

MetodoQuando usarloEsempio didattico
GETLetture, filtri, ricercheelenco.php?classe=5BIA
POSTInvio dati da form, inserimentiform per aggiungere uno studente

Nel modulo useremo soprattutto GET per filtrare e POST per inserire dati nel database.

Se perdi questo schema, poi $_POST, PDO e HTML sembrano argomenti separati. In realta fanno parte dello stesso flusso.

Struttura minima del progetto

  • public/: pagine accessibili dal browser, per esempio index.php e insert.php.
  • config/: parametri di connessione al DB, fuori dalla parte pubblica.
  • src/ o lib/: funzioni riutilizzabili, per esempio apertura della connessione PDO.
  • data/: script SQL di creazione e dati di prova.

L'idea chiave non e la struttura perfetta del progetto, ma separare bene pagine, configurazione e supporto al database.

Micro-demo: PHP dentro HTML

<h1>Elenco studenti</h1>
<?php
$studenti = ["Ada", "Alan", "Grace"];
foreach ($studenti as $nome) {
    echo "<p>$nome</p>";
}
?>

Per ora i dati arrivano da un array. Nelle prossime slide lo stesso meccanismo servira per stampare righe lette da MySQL.

Mappa del modulo

  • Adesso: fissiamo richiesta, risposta, input e ruolo del database.
  • Subito dopo: prepariamo ambiente e connessione PDO.
  • Poi: costruiamo script PHP che leggono input, eseguono query parametrizzate e mostrano risultati.

Se questa mappa e chiara, tutto il resto del modulo diventa piu naturale.
